# AI金融黑金商务视频设计规范

## 设计目的

这是 `jtxvideo-skill` 的默认视频设计规范。每次新口播视频项目开始时，先把本文件复制为项目内的 `design.md`，再根据当期内容做少量调整。

目标不是普通黑金模板，而是适合你的账号定位：

**AI金融博主 / 量化公司AI产品经理 / 投研工具实践者 / 开源社区建设者**

视频应像一份高级 AI 金融投研简报叠加在口播视频上：专业、克制、可信、有信息密度；不要花哨、炫技、带货模板感。

## 基础格式

- 平台：抖音短视频
- 画布：1080x1920 竖屏工程，最终可导出 1440x2560
- 主素材：口播视频
- 主视频：必须使用预处理后的 30fps 固定关键帧 H.264 文件
- 画面原则：口播人物是主体，动效卡片是辅助解释

## 核心视觉方向

- 黑金商务 + AI金融简报
- 深色玻璃质感卡片
- 细金色强调线
- 少量青蓝色 AI / 工具感辅助色
- 信息模块清晰、有层级
- 像投研简报，不像娱乐模板
- 像金融分析内容，不像炒股群信号

## 人物亮度规则

人物必须透亮、自然、可信。禁止为了黑金风格给整条视频盖一层重黑膜。

推荐基础处理：

- 主视频亮度：`brightness(0.92)` 到 `brightness(1.0)`
- 对比度：`contrast(1.0)` 到 `contrast(1.05)`
- 饱和度：`saturate(1.0)` 到 `saturate(1.06)`
- 全局黑色渐变只能轻微使用，不得明显压暗人脸
- 网格/纹理透明度控制在 `0.10` 到 `0.18`

避免：

- `brightness(0.76)` 这类明显压暗人物的处理
- 大面积黑色遮罩覆盖脸部
- 为了卡片可读性牺牲人物亮度

正确做法：

- 人物保持亮
- 卡片自身加深
- 卡片文字加阴影或提高对比
- 不用全局黑膜解决卡片可读性

## 色彩规范

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 主背景黑 | `#050505` | 主画布背景 |
| 深色卡片 | `#0A0B0A` | 主要信息卡片 |
| 次级面板 | `#10120F` | 辅助卡片、侧边模块 |
| 玻璃卡片 | `rgba(14, 15, 13, 0.88)` | 叠加在视频上的卡片 |
| 字幕浅黑 | `rgba(28, 28, 25, 0.52)` | 字幕背景 |
| 弱边框 | `rgba(255, 255, 255, 0.14)` | 普通卡片边框 |
| 金色边框 | `rgba(224, 177, 74, 0.42)` | 重点卡片边框 |
| 主金色 | `#E0B14A` | 核心数据、重点词、价值判断 |
| 暗金色 | `#B88A2D` | 小标签、次级强调 |
| 亮金色 | `#F4D36A` | 极少数关键数字 |
| 主文字 | `#F4F0E6` | 主要文字 |
| 次级文字 | `#DED7C8` | 说明文字 |
| 弱文字 | `#B8B0A2` | 小标签、元信息 |
| 科技蓝 | `#20A7D8` | AI 工具、系统状态 |
| 青色 | `#37D4E8` | 小图标、激活状态点 |
| 风险红 | `#E25555` | 负面对比，少用 |
| 风险琥珀 | `#D9962F` | 待确认、观点判断、风险提示 |

## 色彩使用原则

- 黑色是卡片语言，不是全屏黑膜。
- 金色只用于重点，不做装饰乱飞。
- 蓝色/青色只用于 AI 工具、系统、模型能力相关信息。
- 红色和绿色要克制，不能做成股票软件界面。
- 禁止大面积紫色渐变、蓝紫科技风、米色背景。

## 字体规范

优先使用系统字体，不强制引入外部字体。

| 场景 | 字体建议 | 用法 |
| --- | --- | --- |
| 大数字 | `DIN Condensed`、`Arial Narrow`、`Impact` | 涨幅、百分比、关键数字 |
| 中文标题 | `PingFang SC`、`Heiti SC`、`Arial` | 观点、结论、问题 |
| 正文 | `PingFang SC`、`Arial` | 字幕、说明文字 |
| 工具/模型名 | `SF Mono`、`Menlo` | Claude Code、GLM-5.2、Opus 等 |
| 金融标签 | `SF Mono`、`Menlo` | 日期、百分比、来源、状态标签 |

## 字号建议

| 类型 | 字号 | 用途 |
| --- | --- | --- |
| 超大数字 | 112-156px | 开场涨幅、结尾关键数字 |
| 大标题 | 72-92px | 核心观点 |
| 卡片标题 | 40-56px | 信息卡主标题 |
| 卡片正文 | 28-36px | 简短说明 |
| 标签 | 20-26px | 日期、来源、小分类 |
| 底部字幕 | 44-48px | 口播字幕 |

## 布局规范

安全区：

- 顶部：96px
- 左右：64px
- 底部：抖音发布安全区按 300px 以上预留
- 字幕默认底边：`bottom: 340px` 到 `bottom: 380px`

人物处理：

- 口播人物是情绪锚点，不能被卡片抢走。
- 不遮眼睛、嘴巴、关键表情。
- 字幕可以压到衣服区域，但不要压嘴。
- 右侧和右下角尽量留给抖音 UI。

信息区域：

| 区域 | 用途 |
| --- | --- |
| 左上角 | 小标签、当前段落标识 |
| 上中部 | 核心问题、核心观点 |
| 左中部 | 数据卡、事件卡、对比卡 |
| 右中部 | 工具卡、模型卡、状态卡 |
| 下三分之一 | 字幕、补充说明 |
| 右下角 | 尽量留空 |

## 字幕系统

字幕默认样式：

```css
.full-subtitle {
  left: 64px;
  right: 64px;
  bottom: 360px;
  min-height: 118px;
  padding: 24px 34px;
  border-radius: 8px;
  border: 1px solid rgba(255, 255, 255, 0.14);
  border-left: 6px solid #E0B14A;
  background: rgba(28, 28, 25, 0.52);
  box-shadow: 0 14px 38px rgba(0, 0, 0, 0.26);
  backdrop-filter: blur(6px);
  color: #F4F0E6;
  text-shadow: 0 2px 8px rgba(0, 0, 0, 0.58);
  font-size: 46px;
  line-height: 1.22;
  font-weight: 800;
  text-align: center;
}
```

字幕原则：

- 字幕要大、清楚、像短视频成片。
- 字幕不能太低，避免被账号名、标题、原声和底部导航遮挡。
- 字幕背景不能纯黑不透明，避免挡脸。
- 关键词、数字、模型名可用金色强调。

## 卡片系统

基础卡片：

- 背景：`rgba(14, 15, 13, 0.88)` 到 `rgba(14, 15, 13, 0.92)`
- 边框：`1px solid rgba(255, 255, 255, 0.14)`
- 圆角：最多 8px
- 内边距：28-36px
- 阴影：`0 24px 60px rgba(0, 0, 0, 0.42)`
- 可加很细的金色上边线

重点卡片：

- 只突出一个主要数字或结论
- 内容不超过 3 行
- 金色用于数据和判断，不用于无意义装饰

技术卡片：

- 用青蓝色表示工具、模型、状态
- 蓝色面积不要超过卡片的 15%
- 不做成纯代码教程

## 动效规范

整体动效要像金融简报，不像模板特效。

允许：

- 轻微滑入
- 淡入
- 数据条展开
- 卡片层级切换
- 重点词轻微高亮

禁止：

- 从上到下滚动的金色线条
- 金光乱飞
- 粒子爆炸
- 大面积闪烁
- 赛博朋克霓虹
- 把流程说明做进视频画面，除非视频主题就是教程

## 金融身份表达

视频要体现：

- 你懂市场信号
- 你懂 AI 产品事件如何影响市场叙事
- 你会自己实测 AI 工具
- 你区分得出情绪、证据、投资逻辑
- 你在为 AI 金融开源社区建立长期可信度

避免：

- 喊单感
- 荐股感
- 暴富感
- 暗示买了就赚
- 炒股群信号风

## 输出规格

审核版：

- `1080x1920`
- `30fps` 或 `60fps`
- 完整音频重新封装

最终发布版：

- `1440x2560`
- `60fps`
- H.264 High Profile
- AAC 音频
- `+faststart`

## 最终检查

导出前必须检查：

- 人物是否透亮
- 字幕是否避开抖音 UI
- 字幕是否挡脸
- 卡片是否抢主体
- 动效是否过度
- 音频是否完整
- 最终是否为 2K 60fps
